## Введение <a name="введение"></a>

**Актуальность темы.** В современном мире сельское хозяйство остается одной из самых уязвимых отраслей перед лицом климатических изменений и погодных аномалий. Воздействие таких факторов, как экстремальные температуры, засуха, избыточные осадки и сильные ветра, может привести к значительной потере урожая. Своевременное выявление и оценка рисков позволяют аграриям принимать превентивные меры, снижая экономические потери. В связи с этим разработка доступного, быстрого и интуитивно понятного программного инструмента для оценки агрорисков является актуальной задачей.

**Цель работы** — разработать современное клиентское веб-приложение (SPA) для оценки сельскохозяйственных рисков на основе вводимых климатических параметров.

**Задачи исследования:**
1. Провести анализ предметной области и определить ключевые факторы риска.
2. Выбрать оптимальный стек технологий для быстрой разработки клиентского приложения.
3. Разработать алгоритм оценки уровня риска на основе метеорологических данных.
4. Спроектировать современный пользовательский интерфейс с использованием концепции Bento Grid.
5. Реализовать функционал сохранения истории расчетов, построения графиков и экспорта данных в формат CSV.

**Объект исследования** — процесс прогнозирования рисков при выращивании сельскохозяйственных культур.
**Предмет исследования** — разработка программного обеспечения для автоматизации расчетов уровня риска.

## Глава 1. Теоретические и методологические основы разработки <a name="глава-1-теоретические-основы"></a>

### 1.1. Актуальность прогнозирования рисков в сельском хозяйстве
Сельскохозяйственные культуры, такие как пшеница, кукуруза, картофель и другие, требуют определенных климатических условий для оптимального роста. Отклонения характеристик среды (температура, влажность, скорость ветра, количество осадков) вызывают:
*   **Засуху** — дефицит влаги при высоких температурах, приводящий к гибели корневой системы.
*   **Заболевания растений** — высокая влажность в сочетании с обильными осадками способствует развитию грибковых инфекций.
*   **Механические повреждения** — сильные порывы ветра способны сломать стебли высоких культур (например, кукурузы или подсолнечника).

### 1.2. Выбор стека технологий (React, TypeScript, Tailwind CSS)
Для реализации системы был выбран современный стек фронтенд-разработки:
*   **React (v19):** Библиотека для создания пользовательских интерфейсов. Позволяет разбить приложение на независимые компоненты, что упрощает масштабирование и поддержку.
*   **TypeScript:** Строго типизированное надмножество JavaScript. Использование интерфейсов позволяет минимизировать ошибки на этапе разработки, четко описывая структуры данных (вводимые параметры среды, результаты расчетов).
*   **Tailwind CSS:** Фреймворк на основе utility-классов. Обеспечивает быструю стилизацию прямо в разметке компонентов без необходимости писать объемные CSS-файлы.
*   **Vite:** Современный и быстрый сборщик модулей, обеспечивающий мгновенную горячую перезагрузку (HMR) в процессе разработки.
*   **Lucide React:** Библиотека легковесных SVG-иконок для визуального обогащения интерфейса.

### 1.3. Концепция пользовательского интерфейса (Bento Grid)
Bento Grid — это популярный тренд в веб-дизайне, вдохновленный японскими коробками для еды (бенто). Он предполагает разделение экрана на аккуратные прямоугольные и квадратные секции с закругленными углами, организованные в модульную сетку. 
Для приложения AgroRisk этот подход позволил уместить всю необходимую информацию (форму ввода, карточку результата, график динамики и таблицу истории) на одном экране без необходимости прокрутки (на десктопных устройствах), создавая ощущение профессиональной панели управления (Dashboard).

## Глава 2. Практическая реализация системы «AgroRisk» <a name="глава-2-практическая-реализация"></a>

### 2.1. Описание архитектуры и структуры данных
Архитектура представляет собой Single Page Application (SPA), работающее исключительно на стороне клиента. База данных не используется, что обеспечивает независимость приложения от серверов и высокую скорость работы.

Основные структуры данных (описаны в `src/types.ts`):
*   `AgroData` — входные данные от пользователя (регион, температура, влажность, осадки, скорость ветра, тип культуры).
*   `RiskResult` — результат работы алгоритма (уровень риска, тип угрозы, текстовая рекомендация, цветовой индикатор).
*   `HistoryEntry` — агрегированный объект, сохраняемый в истории логов, включающий в себя данные формы, результат и метаданные (уникальный ID, временная метка).

### 2.2. Разработка алгоритма расчета рисков
Логика оценки рисков инкапсулирована в функции `calculateRisk` (файл `src/lib/riskCalculator.ts`). Алгоритм построен на базе каскадной системы правил (if-else):
1.  **Высокий риск засухи:** если температура > 30°C и осадки < 10 мм.
2.  **Высокий риск заболеваний (грибок/плесень):** если влажность > 80% и осадки > 80 мм.
3.  **Высокий риск повреждения посевов:** если скорость ветра > 15 м/с.
4.  **Средний риск (смешанные факторы):** если любой из параметров близок к критическому (например, температура > 28°C или ветер > 10 м/с).
5.  **Низкий риск:** во всех остальных случаях (оптимальные условия).

Функция возвращает объект с уровнем риска, описанием ситуации и практической рекомендацией для фермера.

### 2.3. Реализация модулей интерфейса
Интерфейс системы разбит на независимые компоненты:
*   `RiskForm.tsx`: Контролируемая форма (Controlled Form) на React для ввода 6 параметров.
*   `RiskResultCard.tsx`: Адаптивная карточка для вывода результата. Меняет цвет (красный, желтый, зеленый) и иконку в зависимости от тяжести риска.
*   `RiskChart.tsx`: Компонент для визуализации последних записей истории в виде графических столбиков, показывающих динамику (рассчитывается высота блока в процентах).
*   `HistoryTable.tsx`: Модуль, выводящий лог расчетов в виде компактного списка.

### 2.4. Организация локального хранения (LocalStorage) и экспорта (CSV)
Поскольку приложение не имеет бэкенда, для сохранения истории между сессиями браузера используется API `window.localStorage`.
*   При первой загрузке данные восстанавливаются из LocalStorage или подставляются из демо-файла `data.json`.
*   При каждом новом расчете массив истории обновляется и синхронизируется с LocalStorage.

**Экспорт в CSV**
Реализована функция `handleExportCSV` в компоненте `HistoryTable`. Алгоритм работы:
1.  Извлечение текущих данных истории.
2.  Формирование строки заголовков.
3.  Преобразование объектов истории в массив строк, разделенных запятыми.
4.  Сборка единого текстового документа формата CSV.
5.  Создание объекта `Blob` (Binary Large Object) с указанием кодировки UTF-8 (а также добавление BOM `\uFEFF` для корректного отображения кириллицы в MS Excel).
6.  Генерация URL через `URL.createObjectURL` и программный клик по невидимой ссылке `<a>` с атрибутом `download`, что вызывает диалоговое окно сохранения файла у пользователя.

## Заключение <a name="заключение"></a>

В ходе выполнения курсовой работы были успешно достигнуты все поставленные цели. Разработано современное, надежное и визуально привлекательное клиентское веб-приложение для прогнозирования рисков в сельском хозяйстве — **AgroRisk**.

Приложение позволяет фермерам и агрономам оперативно оценивать угрозы засухи, заболеваний растений и ветровых повреждений, исходя из текущих или прогнозируемых климатических условий. Использование современного подхода в дизайне (Bento Grid) обеспечило высокую информативность и удобство навигации: вся ключевая информация доступна без смены экранов.

За счет отсутствия сложной серверной инфраструктуры (использован подход Local-First с сохранением данных в браузере), система работает мгновенно и может быть развернута на любом статическом хостинге. Встроенный функционал экспорта CSV позволяет пользователям сохранять свои исторические данные для последующего анализа в табличных процессорах.
